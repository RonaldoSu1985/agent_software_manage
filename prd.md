# 代理商软件库存管理系统 - PRD文档

## 1. 项目概述

### 1.1 项目背景
本系统是一个代理商软件库存管理系统，用于管理代理商的软件库存、采购记录、安装记录、划拨记录以及库存变动记录。系统采用RBAC（基于角色的访问控制）进行权限管理，并支持数据字典配置。

### 1.2 系统目标
- 实现代理商软件库存的统一管理
- 支持采购、安装、划拨等业务操作
- 提供完整的库存变动追溯能力
- 实现角色级别的权限控制
- 通过数据字典实现系统参数的动态配置

### 1.3 功能模块
| 模块 | 描述 |
|------|------|
| 代理商库存管理 | 查看代理商软件库存列表 |
| 代理商采购记录 | 管理代理商软件采购记录 |
| 代理商安装记录 | 管理商户软件安装记录 |
| 代理商划拨记录 | 管理代理商之间的软件划拨 |
| 代理商库存记录 | 查看库存变动明细 |
| 数据字典 | 管理系统参数配置 |
| 用户管理 | 管理系统用户 |
| 角色管理 | 管理角色和权限 |

## 4. 业务流程

### 4.1 采购流程
1. 用户填写采购信息
2. 系统验证代理商和软件信息
3. 创建采购记录
4. 更新库存数量（增加）
5. 记录库存变动日志

### 4.2 安装流程
1. 用户填写安装信息
2. 系统验证代理商、软件和库存
3. 创建安装记录
4. 更新库存数量（减少）
5. 记录库存变动日志

### 4.3 划拨流程
1. 用户填写划拨信息
2. 系统验证划出/划入代理商和软件
3. 验证划出方库存充足
4. 创建划拨记录
5. 更新划出方库存（减少）
6. 更新划入方库存（增加）
7. 记录双方库存变动日志

### 4.4 数据字典配置流程
1. 创建字典类型（如：代理商所属系统、软件名称）
2. 创建字典项（如：V3系统、LTB系统；汇客餐饮、汇客零售）
3. 系统各模块从数据字典读取配置值
